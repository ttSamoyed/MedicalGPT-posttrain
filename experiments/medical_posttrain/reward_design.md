# GRPO Reward Design

## 目标

训练模型在医疗问答中输出更清晰的推理结构和更可靠的行动建议，同时避免 reward hacking。

## Reward Components

### 1. 格式奖励

鼓励输出结构：

```text
<think>简短医学判断依据</think>
<answer>面向患者的回答和建议</answer>
```

注意：如果最终产品不希望暴露推理，可以在数据中把 `<think>` 控制为简短、可审计的医学依据，而不是长链路思维。

### 2. 医学语义相似度

使用轻量 embedding 模型计算回答与参考要点的相似度，例如 `text2vec-base-chinese`。

适合奖励：

- 是否覆盖关键检查
- 是否识别危险信号
- 是否给出合理就医建议

### 3. LLM Judge

使用较小 judge model 给出 1-5 分：

- 专业性
- 安全性
- 是否直接回答问题
- 是否避免夸大和虚假诊断

### 4. PPL 惩罚

用小模型计算生成文本困惑度，惩罚胡言乱语、乱码、重复循环。

## Reward Hacking 风险

- 模型只输出固定模板骗格式分
- 模型堆砌医学术语骗 judge 分
- 模型过度保守，所有问题都回答“请就医”
- 模型生成冗长重复内容降低可用性

## 处理策略

- 格式奖励权重不要过高
- judge prompt 中加入“直接性”和“行动建议”评分
- 对重复 n-gram 加惩罚
- 保留人工案例评估，不只看训练 reward
